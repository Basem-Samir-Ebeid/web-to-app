## FIX 2 — Remove hardcoded HMAC secret (CRITICAL)
**File:** `app/src/main/java/com/webtoapp/core/activation/DeviceIdGenerator.kt`

Replace the hardcoded constant:
```kotlin
private const val HMAC_SECRET = "WTA_DeviceId_Integrity_2024"
```
With a method that generates the secret from the app's own signing signature at runtime using PackageManager, so it is never stored as a plain string in the source code. Use the first signing certificate's SHA-256 bytes as the HMAC key material. Fall back to a UUID stored in EncryptedSharedPreferences if the signature is unavailable.

---

## FIX 3 — Fix decryptStream to use streaming instead of loading full file into RAM (CRITICAL)
**File:** `app/src/main/java/com/webtoapp/core/crypto/EnhancedCrypto.kt`

The current `decryptStream` function calls `input.readBytes()` which loads the entire encrypted file into memory. Rewrite it to use `CipherInputStream` for true streaming decryption, matching the approach already used in `encryptStream`. Remove the `maxDecryptSize` check since streaming no longer has the memory constraint.

## FIX 4 — Restrict cleartext traffic in production (CRITICAL)
**File:** `app/src/main/res/xml/network_security_config.xml`

Change the base-config to deny cleartext traffic by default. Only allow cleartext for localhost and 127.0.0.1 (which are already in the domain-config). The file should become:
```xml
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <base-config cleartextTrafficPermitted="false">
        <trust-anchors>
            <certificates src="system" />
        </trust-anchors>
    </base-config>
    <domain-config cleartextTrafficPermitted="true">
        <domain includeSubdomains="false">localhost</domain>
        <domain includeSubdomains="false">127.0.0.1</domain>
        <trust-anchors>
            <certificates src="system" />
        </trust-anchors>
    </domain-config>
</network-security-config>
```
Note: the debug config at `app/src/debug/res/xml/network_security_config.xml` can remain unchanged.

## FIX 5 — Fix unreliable progress reporting in encryptStream (WARNING)
**File:** `app/src/main/java/com/webtoapp/core/crypto/EnhancedCrypto.kt`

In `encryptStream`, the line:
```kotlin
val totalSize = input.available().toLong()
```
`available()` does not return the full stream size reliably. Add an optional `totalSize: Long = -1L` parameter to the function signature and use it for progress reporting. If `-1`, pass `-1` to `onProgress` as the total to indicate unknown size.

## FIX 6 — Replace finalize() with Cleaner or explicit close() (WARNING)
**File:** `app/src/main/java/com/webtoapp/core/crypto/EnhancedCrypto.kt` and `app/src/main/java/com/webtoapp/core/crypto/SecureMemory.kt`

Remove all `protected fun finalize()` overrides. They are deprecated since Java 9 and not guaranteed to be called. The classes already implement `AutoCloseable` / `Closeable`, so callers should use `use {}` blocks. Add a `@Suppress("ObjectFinalize")` comment where needed and document that callers are responsible for calling `close()`.

## FIX 11 — Remove redundant database index on web_apps table (IMPROVEMENT)
**File:** `app/src/main/java/com/webtoapp/data/model/WebApp.kt`
